# Swaply β1 配送・匿名配送方針

最終更新: 2026-05-28
位置づけ: 今後の実装・UI・DB・ロードマップ判断の正式ドキュメント

> 本書は Swaply β1 における「配送・匿名配送方針」を整理し、今後の実装判断の基準にすることを目的とする。

---

## 1. 結論

Swaply β1 では、**匿名配送 API 連携・送料決済・QR 発行・追跡自動連携は実装しない**。

ただし、**匿名配送を諦めるわけではない**。

匿名配送は Swaply の中核機能であり、X 交換に対する最大級の差別化要素として、β1 時点からロードマップ上で明確に提示する。

### β1 で優先する機能

- 譲求マッチング
- 出品 / 検索
- 交換提案
- 複数枚交換の最低限対応
- Trust 最低限
- 会場交換 / 手渡し交換
- 通常郵送の取引管理
- **匿名配送準備中表示**
- **匿名配送待ち登録**
- 住所共有に関する明確な注意喚起
- 通報 / キャンセルの最低限導線

### 実装タイミング

匿名配送は、配送会社連携・送料決済・QR 発行の見通しが立った段階で、**Phase 1.5 または Phase 2 の最優先機能**として実装する。

---

## 2. なぜ β1 で匿名配送を完全実装しないのか

匿名配送は重要だが、β1 で中途半端に入れると危険。理由は以下。

### 2.1 一部ユーザー限定の匿名配送は初期流動を壊す

匿名配送を一部ユーザーだけに開放すると、以下の問題が起きる。

- A さんは匿名配送が使える
- B さんは匿名配送が使えない
- しかし譲求条件は合っている
- 交換成立後に「この相手とは匿名配送できない」が発生する
- 結局、住所交換が必要になる
- 「匿名配送できると聞いたのに」という期待値ズレが起きる

これは、Swaply の信頼を落とす可能性がある。

**したがって、匿名配送はユーザー単位で中途半端に開放しない。**

### 2.2 完全匿名配送には重い実装が必要

匿名配送を正式に名乗るには、少なくとも以下が必要。

- 配送会社 API 連携
- アプリ内送料決済
- 配送ラベル / QR 発行
- 発送期限管理
- 追跡番号または配送ステータス管理
- 受取確認
- キャンセル / 通報導線
- 配送事故 / 未着 / 破損時の対応設計
- 利用規約 / プライバシーポリシー整備
- 個人情報管理
- 配送会社との契約 / 審査

特にヤマト運輸の配送連携 API や日本郵便の e 発送サービスは、公開情報上、C2C 匿名配送に適した機能を持つが、契約条件・費用・審査・テスト環境・最低出荷量などは問い合わせ必須。

β1 時点で無理に実装すると、スコープ・運用・法務・決済が一気に肥大化する。

### 2.3 匿名配送なしでも β1 で検証できることはある

β1 の目的は、完成版を出すことではなく、以下を検証すること。

- X 交換より譲求マッチングに価値があるか
- 出品 / 求登録が増えるか
- 提案が発生するか
- 複数枚交換に需要があるか
- 会場交換 / 手渡し交換が使われるか
- Trust 表示が安心材料になるか
- 通常郵送でどの程度交換が進むか
- 住所共有が理由でどの程度交換が止まるか
- 匿名配送希望がどれだけ強いか

**匿名配送がないことで止まった取引も、重要な検証データになる。**

## 3. β1 で提供する配送方法

β1 では、配送 / 交換方法を以下のように扱う。

### 3.1 会場交換

ライブ・イベント会場での交換。

- β1 では優先度高
- ただし、高度な Hold 機能、重いペナルティ、複雑な到着確認は後回し

### 3.2 手渡し交換

ユーザー同士が合意した場所での手渡し交換。

- β1 で対応
- 住所や本名を不要にできるため、**匿名配送未実装期間の重要な交換手段**として扱う

### 3.3 通常郵送

β1 では通常郵送を許可する。

ただし、通常郵送は匿名配送ではない。**発送のために、相手へ氏名・住所を共有する必要がある**。

UI 上で必ず明示する。

### 3.4 匿名配送

β1 では**準備中として表示する**。

アプリ内で以下のように扱う。

- 機能としては未提供
- ロードマップ上の最重要項目として表示
- 匿名配送待ち登録を可能にする
- 提供開始時に通知予定
- 配送会社連携の進捗を今後公開する

## 4. β1 の画面 / UX 方針

### 4.1 配送方法選択画面

配送方法選択では、以下の選択肢を表示する。

```text
交換方法を選ぶ

1. 会場で交換
2. 手渡しで交換
3. 郵送で交換(住所共有あり)
4. 匿名配送(準備中)
```

**匿名配送を非表示にしない**。

理由は、ユーザーに「このアプリは住所問題を理解している」と伝えるため。

ただし、匿名配送はまだ使えないことを明確に表示する。

### 4.2 通常郵送の注意モーダル

通常郵送を選択した場合、以下の確認モーダルを表示する。

> **郵送交換について**
>
> 現在、Swaply β1 では匿名配送は準備中です。
>
> 郵送交換を行う場合、発送のために相手へ氏名・住所を共有する必要があります。
>
> 住所を相手に見せたくない場合は、匿名配送の提供開始をお待ちください。

**ユーザーが確認しない限り、通常郵送の取引に進めない**。

### 4.3 チャット内の注意喚起

匿名配送未実装期間は、チャットで以下の注意を表示する。

> 本名・住所・電話番号・LINE ID・SNS 本アカウントの共有は慎重に行ってください。
> 匿名配送は現在準備中です。

**注意点**:

匿名配送未実装期間に「個人情報を送らないでください」と強く書きすぎると、通常郵送が成立しなくなる。

したがって、β1 では「**慎重に共有してください**」という表現にする。

匿名配送実装後は、以下に切り替える。

> 本名・住所・電話番号・LINE ID・SNS 本アカウントは、取引チャットに送らないでください。

### 4.4 匿名配送準備中カード

アプリ内に以下のようなカードを表示する。

> **住所を見せない匿名配送を準備中**
>
> 推し活交換では、本名・住所・LINE・本垢を相手に出したくない方が多くいます。
>
> Swaply では、相手に個人情報を見せずに郵送交換できる匿名配送を準備しています。
>
> β1 ではまず、譲求マッチング・会場交換・通常郵送の取引管理から公開します。
>
> 匿名配送を使いたい方は「待ち登録」してください。

### 4.5 匿名配送待ち登録

「匿名配送を待つ」ボタンを設置する。

登録後の文言:

> 登録ありがとうございます。
>
> 匿名配送の希望数は、配送会社連携・開発優先度の判断に利用します。
>
> 提供開始時にお知らせします。

## 5. β1 で取得すべきデータ

匿名配送未実装でも、以下のデータを取得する。

| メトリクス | 説明 |
|---|---|
| `anonymous_shipping_waitlist_count` | 匿名配送待ち登録数 |
| `postal_trade_started_count` | 郵送交換開始数 |
| **`postal_trade_abandoned_due_to_address_count`** | **住所共有が理由で中断した件数** |
| `anonymous_shipping_required_for_trade_count` | 匿名配送があれば交換したかった件数 |
| `handoff_trade_completed_count` | 手渡し / 会場交換完了数 |
| `match_candidate_count` | 譲求マッチ候補数 |
| `proposal_sent_count` | 提案件数 |
| `proposal_accepted_count` | 承認数 |

特に重要なのは以下。

**`postal_trade_abandoned_due_to_address_count` — 住所共有が理由で中断した件数**

これが一定数出れば、匿名配送導入の強い根拠になる。

## 6. X / 外部発信方針

### β1 で使わない表現

以下の表現は β1 時点では使わない。

- ❌ 匿名で交換できます
- ❌ 住所を出さずに交換できます
- ❌ 完全匿名です
- ❌ 誰にも住所が知られません

理由は、β1 時点では匿名配送が未実装であり、通常郵送では相手に氏名・住所を共有する必要があるため。

### 使ってよい表現

> Swaply は、相手に本名・住所を見せない郵送交換を目指しています。
>
> β1 ではまず、譲求マッチング・会場交換・通常郵送の取引管理を先行公開します。
>
> 住所非公開の匿名配送は、配送会社連携の確認後、最優先で導入予定です。
>
> 匿名配送を希望する方は、事前登録で希望を教えてください。

### より強い X 投稿例

> 正直に言います。
>
> Swaply β1 では、最初から完全な匿名配送までは実装しません。
>
> でも、ここは絶対に逃げません。
>
> 推し活交換で怖いのは、
> 本名・住所・LINE・本垢を相手に出すこと。
>
> だから Swaply では、
> 相手に個人情報を見せずに郵送交換できる匿名配送を、最重要機能として準備しています。
>
> β1 ではまず、
> ・譲求が合う相手を探す
> ・会場 / 手渡し交換をしやすくする
> ・郵送交換の不安を見える化する
> ところから公開します。
>
> 匿名配送を早く使いたい方は、
> 事前登録で「匿名配送を待つ」を押してください。
> 希望者数をもとに、配送会社連携を最優先で進めます。

- API 仕様書の閲覧方法

### 7.2 日本郵便 e 発送サービス / プライバシー配送

確認事項:

- 小規模 C2C 交換アプリでも導入相談可能か
- 個人開発者 / 個人事業主 / 法人設立前で相談可能か
- API 仕様書や開発者向け資料の有無
- テスト環境の有無
- 初期費用 / 月額費用 / 件数課金
- 最低出荷量や審査条件

## 10. 最終方針

Swaply β1 では、**匿名配送 API を無理に実装しない**。

理由は、中途半端な匿名配送が初期流動と信頼を壊す可能性が高いため。

代わりに、β1 では以下を行う。

1. 譲求マッチングで交換相手を見つける価値を検証する
2. 会場 / 手渡し交換を成立させる
3. 通常郵送の取引管理を提供する
4. 住所共有がどれだけ障壁になるかを測定する
5. 匿名配送待ち登録を集める
6. 配送会社連携を同時並行で進める

匿名配送は **Swaply の中核機能**であり、X 交換に対する最大級の差別化要素である。

ただし、**β1 では未実装であることを正直に伝える**。

Swaply は「今すぐ匿名配送できるアプリ」としてではなく、

> **住所を出したくない推し活交換を、本気で解決しにいくアプリ**

として β1 を開始する。

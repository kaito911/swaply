## 0. 思想 (旧 v1 から維持)

Swaply の料金体系は以下の思想を構造的に保証する設計。

### 0.1 Trust 思想との整合
- バッジは信頼証明であって序列ではない
- 信頼を貨幣化しない (Trust 系機能は課金不可)
- ユーザーが「上を目指す」ように誘導しない

### 0.2 交換であり売買ではない
- 「差額」ではなく「調整金」と呼称
- 調整金 ¥1,000 上限で売買への滑り落ちを構造的に防ぐ
- 単価上昇による手数料収益最大化の誘惑を構造で遮断

### 0.3 4 分類ルール
機能を 4 つに分類し、課金可能性を構造的にガード:
- **Trust 系**: 課金不可 (信頼の貨幣化を防ぐ)
- **Match 系**: 課金不可 (出会いの貨幣化を防ぐ)
- **Information 系**: 課金可 (密度可視化、市場情報)
- **Convenience 系**: 課金可 (通知優先度、UI 改善)

### 0.4 β保護の構造
- β期間は完全無料
- FEATURE_FLAGS で誤課金を構造的に防止

### 0.5 v1 からの本質変更
- **「コア機能で月額」モデルを廃止** (Round 2 で「推し活市場で成立事例ゼロ」と確認)
- **完全無料コア + 取引時手数料 + 段階的収益化** に転換
- 月額 Premium / Standard 廃止、代わりに Phase 3 で軽 Plus A/B 検証

## 1. プラン構成 (v2 確定)

### 1.1 全プラン一覧

| プラン | 期間 | 価格 | 対象 |
|---|---|---|---|
| **Free (永久)** | - | ¥0 | 全コア機能、永続無料 |
| **Plus (Phase 3、A/B 検証)** | 月額 | ¥300-400 | Convenience 系一部解放、課金率 1% 未達なら撤退 |
| **Pioneer (50 名限定)** | - | 無料 + 特典 | β1 早期協力者、4 特典 (詳細 §3) |

### 1.2 価格設定の根拠

#### Free (永久) コア機能無料の根拠
- Round 2 検証で「コア機能月額」は推し活市場で前例なし
- トレポータルもコア機能は無料 (¥500 サブスクは周辺機能のみ)
- 完全無料コアで競合 (X 譲求 + トレポータル) と差別化、UGC ベースのバイラル獲得

#### Plus ¥300-400 の根拠 (Phase 3 A/B 検証時)
- Oshibana Premium ¥400 (推し棚デコ等) と同水準
- 「実務系便利機能」に限定 (装飾は採用しない、Swaply 文脈と不一致)
- 課金率 1% 未達なら即撤退 = データ駆動の慎重投入

#### 差額決済 10% の根拠
- 業界水準: メルカリ 10%、Vinted 5% + 固定、Depop 0% (決済 2.9-3.3%)
- Stripe Connect 実コスト 3.6% + α を差し引いた手取り ¥20-100/取引
- ¥1,000 上限の調整金枠で「売買化」を構造的に防止

## 3. 4 分類ルールの厳格適用 (v1 から維持)

### 3.1 課金不可 (絶対)

#### Trust 系
- バッジ表示 (Green / Trial Blue / Blue / Gold Blue)
- バッジ詳細 (取引履歴、ship_rate、trouble_count、6 項目)
- バッジに紐づく信頼性表示
- **根拠**: 信頼を貨幣化すると Trust 思想が崩壊

#### Match 系
- カード検索
- 提案送受信
- 取引成立処理
- 連絡先交換
- 配送情報のやり取り
- **根拠**: 出会いを貨幣化すると交換マーケットの本質が失われる

### 3.2 課金可 (条件付き)

#### Information 系
- 取引履歴の詳細統計 (旧 v1 では一部課金、v2 では Free に格上げ)
- ※ v2 では大半を Free 化、Plus は実務系便利機能のみ

#### Convenience 系
- 通知優先度
- 一括出品 / テンプレ
- 保存検索拡張
- 広告非表示
- **条件**: 機能差ではなく UX 差のみ

### 3.3 v1 からの変更

#### コア機能 Plus 化禁忌 (新規)
以下は v1 で Standard/Premium 課金対象だったが、v2 では **永久無料化**:
- 求商品の優先通知 → Free
- 高度な検索 / フィルタ → Free
- 取引履歴の詳細統計 → Free
- 会場モード優先機能 → Free (戦略上)

理由: Round 3 検証で「これらは Swaply の中核 UX、Pro 化すると交換ツールとしての価値が毀損」と確認。

## 5. 調整金 (差額) の構造 (v1 から維持)

### 5.1 呼称
- 「調整金」(交換のニュアンス、売買化を防ぐ)

### 5.2 上限 ¥1,000 (v1 から維持)
- DB レベルで `adjustment_amount` 値域制約
- フロント入力時バリデーション
- RPC レベル再検証

### 5.3 手数料の計算 (v2 で 5% → 10% に変更)

```
調整金 × 手数料率 10%
ただし手数料 ≤ ¥100 (= ¥1,000 × 10%)
```

つまり、1 取引あたりの手数料収益は **最大 ¥100** (v1 の ¥50 から倍増)。

### 5.4 v1 からの変更根拠
- v1: Standard 5% / Premium 3% の 2 段階
- v2: **一律 10%** (Pioneer 50 名のみ段階逓減)
- 根拠: コア機能月額廃止の代わりに取引時手数料を業界水準 (メルカリ 10%) に揃える

### 5.5 Stripe Connect 収納代行スキーム
- Swaply 自身は資金を保持しない (Stripe が出品者へ直接ペイアウト)
- 預かり期間は短く保つ (取引成立後すみやかに出金)
- 資金移動業登録不要 (メルカリ / Vinted と同じ法務構造)

## 7. 運用ルール

### 7.1 価格変更の手続き
- 価格変更は **3 ヶ月前告知**
- Pioneer 特典は本ルールと独立、特典縮小には Pioneer 個別同意必須 (`pioneer_policy_v1.md` §6.1)

### 7.2 解約・返金
- Plus 月額: 解約即時、返金なし
- 差額決済: 取引キャンセル時は決済キャンセル + 返金

### 7.3 トラブル時の対応
- 取引トラブル: 双方の Trust バッジに反映、カウンセリング機能は無料
- システムトラブル: 該当期間の課金免除

## 付録 A: 4 分類ルール判定フロー (v1 から維持)

```
新機能を追加したい
  ↓
この機能は信頼に関わるか? → Yes → Trust 系 (課金不可)
  ↓ No
この機能はマッチング/取引に関わるか? → Yes → Match 系 (課金不可)
  ↓ No
この機能はコア機能か (出品/検索/通知/Trust/会場)? → Yes → Free 永久 (Pro 化禁忌)
  ↓ No
この機能は情報の可視化か? → Yes → Information 系 (Phase 3 Plus 候補)
  ↓ No
この機能は UX 改善か? → Yes → Convenience 系 (Phase 3 Plus 候補)
  ↓ No
分類不能 → 4 分類ルール検討委員会で判定
```

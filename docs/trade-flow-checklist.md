> repo 現物ベース作成。実装変更なし。

## 前提: テスト環境の準備

| 必要なもの | 内容 |
|---|---|
| アカウント A | ①レーンを確認する提案送信側 |
| アカウント B | 提案受信側（A が欲しいカードの所有者） |
| A のアクティブカード | `status='active'` のカードが1枚以上 |
| B のアクティブカード | `status='active'` のカードが1枚以上（これが A の交換対象） |
| ①レーン RPC データ | `get_best_trade_candidate` が A に対してデータを返す状態 |

### ① ①レーン表示確認

**操作**: A でログイン → ホームタブを開く

**確認ポイント**

- [ ] `BestTradeCandidateCard` が画面上部に表示される
- [ ] カード画像（`target_card.image_url`）が表示される。`null` の場合は灰色プレースホルダー
- [ ] `title`（小見出し文言）が表示される
- [ ] `target_card.name`（カード名）が 18px 太字で表示される
- [ ] `target_user.display_name`（相手ユーザー名）が表示される
- [ ] `label` バッジが表示される（`almost_confirmed`=「ほぼ確定」緑 / `likely`=「成立しやすい」紫）
- [ ] `reasons` が最大3件表示される（✓ 付き）
- [ ] CTA ボタン（`data.cta` の文言）が表示される
- [ ] サブ CTA「ほかの候補を見る」が表示される

**判定基準**

| 正常 | 異常 |
|---|---|
| カード名・ユーザー名が正しく表示 | ①レーン自体が表示されない（RPC エラー or データなし） |
| `label` バッジが適切な色で表示 | reasons が4件以上表示される |
| ホームタブ再フォーカスで再取得される | ローディングが終わらない |

**注意**: ①レーンが表示されない場合は `get_best_trade_candidate` の RPC エラーをコンソールで確認（`console.error('[home] get_best_trade_candidate', ...)`）

### ② ①レーンから提案送信

**操作**: ①レーンの CTA ボタンを押す

**確認ポイント（offer/create.tsx への遷移）**

- [ ] `offer/create` 画面に遷移する
- [ ] 交換対象カード（`target_card`）が画面に表示されている
- [ ] 「あなたのカード」に `my_card`（①レーンが推薦したカード）が初期選択されている
  - ※ `myCardId` param 経由で pre-select される
- [ ] 対象カードの配送方法が表示される（`shippingLabel()`）
  - 例: 「郵送可・手渡し可」「郵送可・手渡し不可」など
- [ ] 差額条件が表示される（`adjustmentLabel()`）
  - 例: 「差額なし」「差額あり（上限 ¥1,000）」など
- [ ] 利用規約チェックボックスが未チェック状態
- [ ] 「送信する」ボタンが **非活性**（`canSend = false`）

**送信手順**

1. my_card が正しいことを確認（必要なら変更可）
2. 利用規約チェックボックスにチェック
3. （差額ありの場合）差額入力と差額チェックボックスにチェック
4. 「送信する」ボタンが活性化されることを確認
5. ボタンを押す

**確認ポイント（送信後）**

- [ ] Alert 等が出ずに成功画面に遷移する
- [ ] 成功画面に「🎉 提案を送りました！」が表示される
- [ ] 相手のハンドルが `@handle の返答を待っています` で表示される
- [ ] 「ホームに戻る」ボタンでホームに戻れる

**判定基準**

| 正常 | 異常 |
|---|---|
| 成功画面に遷移 | Alert「この出品は現在提案できません」→ target_card が active でない |
| my_card が①レーン推薦カードで初期選択 | my_card が①レーン推薦と異なるカードで初期選択される |
| 「ホームに戻る」で `/(tabs)/home` に戻る | Alert「自分の出品には提案できません」→ A が B のカードを所有している |

**既知の未接続**: `adjustmentPayer`（支払者選択）は UI に表示されるが `createOffer` に渡されない（TODO コメントあり）

- [ ] あなたの発送状況が「発送済み」に変わる

  - 本文: 「追加発送や自己判断のキャンセルは避けてください。必要情報を整理し、運営判断前提で進めます。」

- [ ] 問題報告セクションも消える（`canOpenDispute = false` に）

**canOpenDispute の条件（コードより）**:
```
trade が存在
AND currentUserId != null
AND trade.status NOT IN (completed, cancelled)
```

**判定基準**

| 正常 | 異常 |
|---|---|

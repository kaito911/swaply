## 1. 制度の目的

β1 初期に Swaply の交換文化形成に協力したユーザーに対し、感謝と長期インセンティブを提供する。

**位置づけ**:
- Trust 系の信頼指標とは独立した「歴史的貢献者の記録」
- 序列化ではなく、創業期協力者への謝意
- 段階逓減により長期的な機会損失を構造的に防止

---

## 2. 採用基準

### 2.1 採用枠
- **総枠 50 名** (希少性最大化のため少数精鋭)
  - β1 早期参加: **先着 30 名**
  - 改善協力推薦: **運営推薦 20 名**

### 2.1.1 50 名枠の根拠
- 機能追加要望をくれる現実的な人数 = 30-50 名で十分
- 200 名は多すぎる (累計損失リスク + 希少性希薄化)
- 50 名なら希少性最大、運営工数最小、Pioneer バッジの価値最大化

### 2.2 採用条件
以下全てを満たすこと:
1. **β1 参加**: 初期テスター登録 (Pioneer 申請含む)
2. **実取引 / 実出品**: 出品 3 件以上 **or** 交換 1 件以上
3. **改善協力**: 要望 / バグ報告 / アンケート回答のいずれかに参加
4. **規約遵守**: Swaply 利用規約・本規約を遵守

### 2.3 採用フロー
1. β1 参加時に Pioneer 申請フォームを表示 (任意)
2. 申請者は条件達成 (2.2) で自動採用候補に
3. 運営が月次レビューで承認 (推薦枠 50 名は別途運営判断)
4. 採用通知 → Pioneer バッジ付与

---

## 3. 特典

### 3.1 特典 4 種

| 特典 | 期間 | 内容 |
|---|---|---|
| **Pioneer 称号 + 公式紹介** | **永久** | UI バッジ表示 + 公式 SNS で名指し紹介 |
| **広告非表示** | **提供期間中有効** | 通常広告非表示 (詳細 3.2 参照) |
| **差額決済手数料免除** | **段階逓減** | Y1-Y3: 100% 免除 / Y4: 5% / Y5: 2.5% / Y6+: 通常 10% |
| **Phase 3 Plus 無料** | **3 年限定** | Phase 3 開始から 3 年間、Plus 機能フル解放 |

### 3.2 「広告非表示」の対象外

以下は Pioneer であっても表示される:
- スポンサー表示 (公式パートナー協賛)
- 法令上必要な表示 (景表法等の遵守表示)
- サービス運営上必要な告知 (システムメンテナンス等)
- 会場協賛枠 (ライブ会場・主催者協賛)

### 3.3 段階逓減方式の意図

差額決済手数料免除を「永久 0%」ではなく段階逓減とする理由:
- Pioneer 50 名 × 永久 0% = 累計機会損失 ¥7.2 万 (Y1-Y5、200 名旧案 ¥240 万から圧縮)
- 段階逓減 + 50 名上限により累計損失を **満額 ¥4.5 万 / 実損 ¥1.35 万** (継続率 30% 込み) に圧縮
- 創業期感謝の意図 (Y1-Y3 完全免除) を維持しつつ、長期的な事業持続性を確保
- Base 累計利益 ¥30-80 万 への影響は無視可能

## 4. 失効条件

以下のいずれかに該当した場合、Pioneer 資格は失効する:

- 不正利用 (アカウント複数取得、なりすまし等)
- 転売目的の利用
- Swaply 利用規約違反
- 未発送 / 受取確認忌避が継続的に発生
- 迷惑行為 (他ユーザーへのハラスメント等)
- 詐欺疑い (Trust 系トラブル多発)

### 4.1 再付与の禁止
失効した Pioneer 資格は**再付与しない**。

---

## 5. 失効通知プロセス

3 段階のアプリ内通知を実施:
1. **失効 90 日前**: 警告通知 + 該当行為の改善案内
2. **失効 30 日前**: 最終警告通知
3. **失効当日**: 失効確定通知 + 特典の段階的停止

異議申し立ては失効 90 日前通知から 30 日以内に運営へ。

---

## 6. 規約変更条項

### 6.1 Pioneer 特典の縮小について
Pioneer 特典の縮小・廃止には **Pioneer 個別の同意なしで適用しない**。

これは `pricing_v2.md §7.1` (価格変更 3 ヶ月前告知) と独立した条項であり、Pioneer 特典は強い保護下にある。

### 6.2 規約全体の改定
本規約の改定は 3 ヶ月前にアプリ内 + メールで告知。Pioneer 特典の縮小を伴う改定の場合、6.1 の同意要件が優先される。

## 7. Trust 思想との関係

### 7.1 序列化の否定
Pioneer バッジは「貢献の事実記録」であり、ユーザー間の序列化を意図しない。

### 7.2 Trust factual metrics との独立
Trust 系 (ship_rate、reply_median_hours、trouble_count 等) は信頼指標。Pioneer は歴史的貢献者の記録であり、両者は独立した次元。

### 7.3 「越えられない壁」リスクへの配慮
Pioneer 50 名以降の新規ユーザーが「Pioneer になれなかった負け組」と感じるリスクを軽減するため:
- Pioneer 紹介ページで「β1 期間限定の創業期協力者 (50 名上限)」と明示
- Phase 4 以降は別の長期貢献者制度を検討 (例: Veteran 制度)
- 50 名という小さな枠は「特別な少数」として位置付けやすく、後発ユーザーへの心理影響を最小化

## 付録 B: 段階逓減の試算根拠 (Pioneer 50 名前提)

| 項目 | 数値 |
|---|---|
| Pioneer 上限 | **50 名** (β1 先着 30 + 運営推薦 20) |
| 平均 ARPU (差額決済) | ¥20/月 |
| Y1-Y3 累計 (100% 免除、36 ヶ月) | 50 × ¥20 × 36 = **¥3.6 万** |
| Y4 累計 (5% 免除、12 ヶ月) | 50 × ¥20 × 0.5 × 12 = **¥0.6 万** |
| Y5 累計 (2.5% 免除、12 ヶ月) | 50 × ¥20 × 0.25 × 12 = **¥0.3 万** |
| **満額累計 (Y1-Y5)** | **¥4.5 万** |
| 5 年継続率 30% 想定での実損失 | **¥1.35 万** |

### 旧案 (200 名上限) との比較
| ケース | 満額累計 | 実損 (継続率 30%) |
|---|---|---|
| 200 名 + 5 年永久 0% | ¥240 万 | ¥72 万 |
| 200 名 + 段階逓減 | ¥147.6 万 | ¥44.3 万 |
| **50 名 + 段階逓減 (確定案)** | **¥4.5 万** | **¥1.35 万** |

200 名上限 → 50 名上限 への変更により、累計機会損失を **97%+ 圧縮**。Base 利益 ¥30-80 万 への影響は無視可能。

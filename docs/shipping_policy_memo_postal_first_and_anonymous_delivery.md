# Swaply 配送方針メモ：郵便起点と匿名配送検証

- 作成日: 2026-05-30
- 目的: 推し活ランダムグッズ交換アプリ「Swaply」における、β1〜β1.5 の配送 UX 方針を整理する。
- 位置づけ: 検証メモ (docs/beta1_shipping_and_anonymous_delivery_policy.md の補強)。来週のオタクユーザーヒアリング結果を追記する前提。

---

## 1. 結論

Swaply の配送 UX は、**ヤマト起点ではなく郵便起点で設計する**。

ヤマト運輸の「宅急便をスマホで送る」には個人間匿名配送の可能性があるが、推し活グッズ交換の実態とはズレる可能性がある。

理由は、Swaply の主対象である以下のグッズは、小さく薄いものが多く、郵便系で送れてしまうため。

- トレカ
- 生写真
- ラントレ
- 特典
- 小さめの缶バッジ
- 一部アクスタ
- 紙類グッズ

そのため、**短期の標準配送 UX は郵便系を前提**にし、**ヤマト匿名配送は「住所を見せたくない人向けの安心オプション」として検証**する。

---

## 2. 現時点の仮説

推し活・オタ活の個人間交換では、実際には以下の配送手段が多い可能性が高い。

**想定順**:

1. 普通郵便 / 定形郵便 / 定形外郵便
2. ミニレター
3. クリックポスト
4. レターパックライト
5. ゆうパケット系
6. ヤマト系

特にトレカ・生写真・ラントレは薄いため、ユーザー心理としては以下になりやすい。

> - わざわざ宅急便？
> - 高くない？
> - コンビニに持っていくの面倒では？
> - 郵便でよくない？

---

## 3. ヤマト匿名配送の位置づけ

ヤマト運輸の「宅急便をスマホで送る」には、個人間で住所・氏名を直接知らないまま送れる**匿名配送の可能性**がある。

ただし、Swaply での位置づけは「**標準配送ではなく、以下のような場面の安心オプション**」とする。

- 住所を絶対に見せたくない
- 初回取引で不安が強い
- 高額グッズを交換する
- ぬい・アクスタ・複数グッズなど、やや大きいものを送る
- 通常郵送では不安がある

つまり、短期的には以下の位置づけ。

| 配送方法 | 役割 |
|---|---|
| **通常郵送** | 標準 |
| **手渡し / 会場交換** | イベント時の即時交換 |
| **ヤマト匿名配送** | 住所を見せたくない人向けの安心オプション |

---

## 4. β1 の配送方針

β1 では、**匿名配送 API 連携は実装しない**。

ただし、郵送交換には**住所共有が必要であることを明示**する。

### β1 で明示すべきこと

> 郵送交換では、発送のために氏名・住所の共有が必要です。
> 住所を見せずに送れる方法も検証中です。

または、少し丁寧にする場合:

> 郵送交換では、普通郵便・クリックポスト・レターパックなどを使う場合、
> 発送のために相手へ氏名・住所を共有する必要があります。
> 住所を相手に見せたくない場合は、匿名配送の提供開始をお待ちください。

アプリ内では長すぎると重いため、**短い文言を優先**する。

### β1 でやること

- 通常郵送を許可する
- 手渡し / 会場交換を許可する
- 郵送交換では住所共有が必要であることを明示する
- 「匿名配送は準備中」または「住所を見せずに送れる方法も検証中」と表示する
- 匿名配送 API 連携はしない
- 配送会社 URL 自動生成はしない
- 追跡自動連携はしない

---

## 5. β1.5 の候補

β1.5 では、ヤマト匿名配送を**「外部手順モード」として検証する可能性**がある。

### 想定フロー

```text
交換成立
  ↓
配送方法で「ヤマト匿名配送を使う」を選ぶ
  ↓
A がヤマト上で匿名配送 URL を作成
  ↓
A が Swaply 内に URL を貼る、または相手へ共有
  ↓
B が URL から住所入力
  ↓
B もヤマト上で匿名配送 URL を作成
  ↓
B が Swaply 内に URL を貼る、または相手へ共有
  ↓
A が URL から住所入力
  ↓
双方が発送
  ↓
追跡番号を Swaply に登録
  ↓
双方受取で完了
```

### β1.5 で確認すべきこと

- URL を Swaply 内に保存してよいか
- URL の有効期限
- URL の再利用可否
- LINE 外共有が公式に問題ないか
- クロネコメンバーズ登録要否
- 受取側の登録要否
- 着払い不可などの制限
- トレカ・生写真など小物交換に送料が見合うか
- 双方が URL 作成する手間を許容できるか
- 追跡番号をユーザー手入力にするか

## 6. 長期方針

長期的な本命は、ヤマトではなく **日本郵便系の匿名小物配送連携** の可能性が高い。

**理由**:

- トレカ・生写真・紙類・ラントレは郵便サイズと相性が良い
- ユーザーの既存習慣も郵便寄りの可能性が高い
- 送料の納得感が出やすい
- ポスト投函・郵便受け配達の心理的負担が少ない

ただし、日本郵便系の匿名配送は、メルカリ・BOOTH 等のプラットフォーム連携前提の可能性があるため、Swaply 単体で短期実装できるとは限らない。

長期的には以下を検討する。

- 日本郵便系配送サービスとの法人連携
- ゆうパケット系匿名配送の可否
- プラットフォーム連携条件
- 追跡連携
- 宛名非表示
- 小物グッズ特化の配送 UX

## 8. 現時点の実装方針

次の配送 UI Phase 1 では、以下の方針にする。

- 郵送 = 住所共有あり
- 匿名配送 = 準備中
- 住所を見せずに送れる方法 = ヤマト等で検証中
- 標準配送 UI = 郵便系の利用実態に合わせる

### 出品詳細 / CTA 付近の短い注意文候補

> 郵送交換では、発送のために氏名・住所の共有が必要です。
> 住所を見せずに送れる方法も検証中です。

### 「匿名配送準備中」文言候補

> 住所を見せずに送れる方法も検証中

または

> 匿名配送は準備中です

前者の方が、ヤマト・日本郵便・将来の法人連携を含めやすい。

---

## 9. 注意点

- ヤマト匿名配送は有望だが、**標準配送にしない**
- 実際の配送習慣は来週のオタクユーザーヒアリングで確認する
- 郵便系の実利用が強ければ、**郵便起点の UI** に寄せる
- ヤマトは安心オプションとして扱う
- β1 で**匿名配送を過剰に約束しない**
- 「匿名配送できます」と断言しない
- 「住所を見せずに送れる方法も検証中」程度に留める
- API 連携や自動 URL 生成は未確認なので実装前に追加調査が必要

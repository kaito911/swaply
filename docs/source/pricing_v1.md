## 0. 思想

Swaply の料金体系は以下の思想を構造的に保証する設計。

### 0.1 Trust 思想との整合
- バッジは信頼証明であって序列ではない
- 信頼を貨幣化しない（Trust 系機能は課金不可）
- ユーザーが「上を目指す」ように誘導しない

### 0.2 交換であり売買ではない
- 「差額」ではなく「調整金」と呼称
- 調整金 ¥1,000 上限で売買への滑り落ちを構造的に防ぐ
- 単価上昇による手数料収益最大化の誘惑を構造で遮断

### 0.3 4分類ルール
機能を4つに分類し、課金可能性を構造的にガード：

- **Trust 系**: 課金不可（信頼の貨幣化を防ぐ）
- **Match 系**: 課金不可（出会いの貨幣化を防ぐ）
- **Information 系**: 課金可（密度可視化、市場情報）
- **Convenience 系**: 課金可（通知優先度、UI 改善）

### 0.4 β保護の構造
- β期間は完全無料
- FEATURE_FLAGS で誤課金を構造的に防止
- 初回成立後課金、市場5条件達成まで課金フラグ OFF

## 1. プラン構成

### 1.1 全プラン一覧

| プラン | 期間 | 価格 | 差額手数料 | 主な機能 |
|---|---|---|---|---|
| Free | - | ¥0 | 5% | 基本機能のみ |
| Standard 単月 | 1ヶ月 | ¥300 | 5% | Information 系一部解放 |
| Standard 3ヶ月パス | 3ヶ月 | ¥800 | 5% | 同上、月平均¥267 |
| Standard 通年 | 1年 | ¥3,000 | 5% | 同上、月平均¥250 |
| Premium 3ヶ月パス | 3ヶ月 | ¥1,500 | 3% | Convenience 系全解放 |
| Premium 通年 | 1年 | ¥6,000 | 3% | 同上、月平均¥500 |

### 1.2 価格設定の根拠

#### Standard 月¥300 の根拠
- K-POPファンの「お小遣い感覚」上限
- カフェ1回分相当
- ライト層が躊躇なく払える水準

#### Premium 通年¥6,000 の根拠
- 月¥500 = ライブ1回分の交通費以下
- 通年契約で「アプリを使い続ける」コミット形成
- ヘビーユーザーの平均利用額に整合

#### 差額手数料 5% / 3%
- Standard 5% = 業界平均
- Premium 3% = 上位プラン優遇、解約防止
- 上限¥1,000 で絶対的キャップ（売買化を防ぐ）

## 2. プラン別機能

### 2.1 Free（無料）

#### 利用可能
- カード出品（無制限）
- 提案送受信
- Trust バッジ（Green / Trial Blue / Blue / Gold Blue 全段階表示）
- 取引成立後の連絡先交換
- 基本通知
- 取引履歴閲覧

#### 制限
- Information 系：詳細インサイト非表示
- Convenience 系：通知優先度なし、UI 簡素

### 2.2 Standard

#### Free に追加
- **Information 系（一部）**:
  - 自分の出品カードの「閲覧数」「提案数」表示
  - 過去30日のトレンド表示

#### Standard 共通
- 差額手数料 5%（上限¥1,000）

### 2.3 Premium

#### Standard に追加
- **Information 系（全解放）**:
  - 市場全体の「需給バランス」可視化
  - 推奨価格レンジ表示
  - 過去90日トレンド
- **Convenience 系（全解放）**:
  - 通知優先度（即時通知）
  - UI カスタマイズ
  - 高度な検索・フィルタ

#### Premium 共通
- 差額手数料 3%（上限¥1,000）

## 3. 4分類ルールの厳格適用

### 3.1 課金不可（絶対）

#### Trust 系
- バッジ表示（Green / Trial Blue / Blue / Gold Blue）
- バッジ詳細（取引履歴、ship_rate、trouble_count）
- バッジに紐づく信頼性表示

**根拠**: 信頼を貨幣化すると Trust 思想が崩壊する

#### Match 系
- カード検索
- 提案送受信
- 取引成立処理
- 連絡先交換
- 配送情報のやり取り

**根拠**: 出会いを貨幣化すると交換マーケットの本質が失われる

### 3.2 課金可（条件付き）

#### Information 系
- 閲覧数・提案数の可視化
- 市場トレンド情報
- 推奨価格レンジ

**条件**: 取引可否に直接影響しない情報のみ

#### Convenience 系
- 通知優先度
- UI 改善
- 検索フィルタ高度化

**条件**: 機能差ではなく UX 差のみ

## 4. β期間中の運用

### 4.1 β期間の料金

- **完全無料**（全プラン解放）
- 全機能を Free として提供
- 課金フラグ（BILLING_ENABLED）は OFF
- 誤課金リスクを構造的に排除

### 4.2 β終了条件（市場5条件）

以下を全て満たした時点で課金開始判断：

1. **DAU**: 100以上が安定
2. **取引成立数**: 月間500件以上
3. **Trust バッジ取得者**: Blue以上が10名以上
4. **重大バグ**: 月1件以下
5. **主催者**: 5名以上が継続使用

### 4.3 課金開始の段階性

#### Phase 1: 完全無料（β）
- 全機能解放
- データ蓄積期間

#### Phase 2: Standard のみ課金開始
- Standard プランのみ提供
- Premium は引き続き無料解放
- 市場反応の観察

#### Phase 3: Premium 課金開始
- 全プラン提供
- 通常運用開始

## 5. 調整金（差額）の構造

### 5.1 呼称の変更
- **旧呼称**: 「差額」
- **新呼称**: 「調整金」

**根拠**: 「差額」は売買のニュアンスを帯びる、「調整金」は交換のニュアンスを保つ

### 5.2 上限 ¥1,000

#### 構造的保証
- DB レベルで `adjustment_amount` の値域制約
- フロントで入力時にバリデーション
- RPC レベルで再検証

#### 上限の意味
- これ以上の差額が出る取引は「交換」ではなく「売買」とみなす
- Swaply は売買マーケットではない、交換マーケットである
- 単価を上げて手数料収益を最大化する誘惑を構造的に遮断

### 5.3 手数料の計算

```
調整金 × 手数料率（Standard 5% / Premium 3%）
ただし手数料 ≤ ¥50（=¥1,000 × 5%）
```

つまり、1取引あたりの手数料収益は最大 ¥50。

## 6. 収益モデルの試算

### 6.1 Year 1（β期間中心）

- 課金収益: ¥0
- 投資: 開発のみ（副業時間）
- 損益: マイナス（運営費のみ）

### 6.2 Year 2（β終了、Standard 課金開始）

#### 想定値（並成功シナリオ）
- DAU: 500
- 月間取引数: 2,000件
- Standard 加入率: 20%（=100名）
- Standard 月収: ¥30,000
- 調整金手数料: 2,000 × ¥30 = ¥60,000
- 月収合計: ¥90,000
- 年収: ¥1,080,000

### 6.3 Year 3（Premium 課金開始、横展開開始）

#### 想定値（並成功シナリオ）
- DAU: 2,000
- 月間取引数: 10,000件
- Standard 加入率: 15%、Premium 加入率: 5%（合計400名）
- Standard 月収: ¥45,000
- Premium 月収: ¥50,000
- 調整金手数料: 10,000 × ¥35 = ¥350,000
- 月収合計: ¥445,000
- 年収: ¥5,340,000

#### 想定値（成功シナリオ）
- DAU: 5,000
- 月間取引数: 25,000件
- 加入率全体: 25%（=1,250名）
- 月収合計: ¥1,500,000
- 年収: ¥18,000,000

#### 想定値（大成功シナリオ）
- 横展開（INI、BABYMONSTER、アニメ等）含む
- 年収期待値: ¥50,000,000+

### 6.4 期待値（確率分布加重）

| シナリオ | 確率 | Year 3 年収 |
|---|---|---|
| 失敗 | 5-8% | ¥0〜200万 |
| 並成功 | 30-35% | ¥500〜1,500万 |
| 成功 | 40-42% | ¥1,500〜5,000万 |
| 大成功 | 17-23% | ¥5,000万〜2億 |

**期待純利益（Year 3）**: 約¥8,500万〜¥1.05億

## 7. 運用ルール

### 7.1 価格変更の手続き
- 価格変更は3ヶ月前告知
- 既存契約者は契約期間終了まで旧価格
- 通年契約者の保護を優先

### 7.2 解約・返金
- 単月: 解約即時、返金なし
- 3ヶ月パス: 解約即時、未経過分返金なし
- 通年: 解約即時、未経過月分の返金あり（手数料¥500差し引き）

### 7.3 トラブル時の対応
- 取引トラブル: 双方の Trust バッジに反映、ただしカウンセリング機能は無料
- システムトラブル: 該当期間の課金免除

## 付録 A: 実装上のフラグ

### FEATURE_FLAGS（H5 で実装済）

```typescript
// constants/feature-flags.ts
export const FEATURE_FLAGS = {
  BILLING_ENABLED: false,        // 課金機能の解放
  PREMIUM_TRIAL_ENABLED: false,  // Premium お試し期間
  DEV_FEATURES: __DEV__,         // 開発用機能
}
```

### β期間中の挙動
- BILLING_ENABLED: false → 課金画面・決済処理を完全無効化
- PREMIUM_TRIAL_ENABLED: false → 全機能解放（Premium 相当）
- DEV_FEATURES: 本番では false、開発時のみ true

### β終了時の解放手順
1. 市場5条件達成を確認
2. BILLING_ENABLED: true に変更
3. Standard プランのみ表示（Premium は隠す）
4. 1ファイル変更で段階的解放

## 付録 B: 4分類ルール判定フロー

新機能追加時の判断フロー：

```
新機能を追加したい
  ↓
この機能は信頼に関わるか？ → Yes → Trust 系（課金不可）
  ↓ No
この機能はマッチング/取引に関わるか？ → Yes → Match 系（課金不可)
  ↓ No
この機能は情報の可視化か？ → Yes → Information 系（課金可）
  ↓ No
この機能は UX 改善か？ → Yes → Convenience 系（課金可）
  ↓ No
分類不能 → 4分類ルール検討委員会で判定
```

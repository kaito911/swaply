# Agent Execution Flow

## 0. 目的

この文書は、SwaplyにおけるAI実装エージェントの実行手順を固定するための文書である。

AIはこの順番以外で動いてはならない。

---

## 1. 入力

人間は以下をAIに渡す。

- タスク名
- 目的
- 完了条件
- 変更してよい範囲
- 変更してはいけない範囲
- UI正本参照先
- 必要なら関連画面名

---

## 2. AIの初動

AIは最初に以下を行う。

1. repo構造確認
2. 対象ファイル候補抽出
3. 関連ファイル抽出
4. ルーティング確認
5. 型定義確認
6. 使用中コンポーネント確認
7. 既存テスト確認
8. CI対象確認

---

## 3. 実装前チェック

AIは実装前に以下を必ず確認する。

- UIは仕様正本に一致しているか
- 既存コードはどこが正本とズレているか
- DB正本に関係するか
- 状態遷移に影響するか
- 危険領域に波及するか
- 変更対象はフロント内で閉じるか

---

## 4. 実装

AIは以下のみを行う。

- 対象ファイル編集
- 関連ファイル編集
- 型修正
- import修正
- テスト修正または追加
- ドキュメント更新

AIは実装中に仕様変更をしてはならない。

---

## 5. 検証

AIは実装後に以下を順番に行う。

1. format
2. lint
3. typecheck
4. test

失敗時はログを読み、原因を特定し、再修正する。

---

## 6. Git処理

検証完了後、AIは以下を行う。

1. branch確認
2. diff確認
3. commit message 作成
4. PR title 作成
5. PR body 作成
6. PR作成

main / master へ直接pushしてはならない。

---

## 7. 人間への出力

AIは最後に以下を人間へ報告する。

- 変更概要
- 変更ファイル一覧
- 影響範囲
- 実行したコマンド
- lint結果
- typecheck結果
- test結果
- 実機確認項目
- 未確定事項
- マージ前に人間が見るべき点

---

## 8. 人間の作業

人間は以下を行う。

- PR確認
- 実機確認
- UI/UX最終判断
- Supabase本番反映判断
- RLS変更判断
- merge判断

---

## 9. 停止条件

以下の場合、AIはPR作成前に停止する。

- migrationが必要
- RLS変更が必要
- 仕様書に矛盾がある
- DB列追加が必要
- CIが安定しない
- 画面変更が危険領域に波及する

---

## 10. 禁止事項

- 差分だけを人間に渡すこと
- 仕様未確定のまま実装すること
- 実機確認を飛ばすこと
- CI未通過でPRを出すこと
- UIを自己判断で改善すること

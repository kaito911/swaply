### 1.2 ARPU 想定 (Y3 末時点、月額)

### 1.3 ARPU 設計根拠

#### 差額決済 ¥15-20 (Base)
- 取引/DAU 比率 = 0.3-0.5 (月)、¥30/取引 (10% × 平均調整金 ¥300) で計算
- Round 4 で「現実 ¥10-20」と指摘、Base は上限寄りだが妥当

#### 広告 ARPU ¥7-10 (Base)
- 日本 AdMob バナー eCPM ¥100-300、推し活ニッチで広告主付きにくい
- Round 4 で「現実 ¥5-10」と指摘、Base は妥当範囲

#### Plus 月収 (Phase 3+)
- 課金率 1% × ¥300-400/月 × DAU で算出
- Base DAU 2,500 × 1% × ¥350 = ¥8,750/月 → ¥0-10,000 レンジ

## 5. 旧 PL からの主な変更点 (Round 1-5 統合)

### 5.1 旧 pricing_v1 PL (Year 3 期待値 ¥8,500 万-1.05 億) からの修正
- **DAU 2,000-5,000 の楽観値を「並成功」扱いしていた** → Base 修正
- **加入率 15-25% (Standard+Premium) を前提していた** → コア機能月額廃止で前提崩壊
- **月額 ARPU + 調整金手数料の二本柱** → 差額決済 + 広告 + Plus + B2B の四本柱に再設計

### 5.2 claude.ai PL v2 (Worst ¥359 万 / Base ¥3,467 万 / Best ¥7.65 億) からの修正
- **Base Y3 DAU 16 万 が過大** → 副業個人開発で達成不可、Target Base 5,000 にダウンサイズ
- **広告 ARPU ¥36-40 が casual game 前提値** → Swaply UX (Aēsop 系) で実態 ¥10-30
- **3 シナリオの幅広すぎ** → 4 シナリオ (Worst/Base/Target Base/Best) で現実的なグラデーション

### 5.3 Round 4 PL v3 (Base ¥250 万) → Round 5 PL v5.2 (Base ¥30-80 万)
- ARPU を下方修正したのに DAU 5,000 維持 = 数値遊び指摘 (Round 5)
- DAU と ARPU の両方を整合させて Base = DAU 2,500-3,000 / 月収 ¥55-75k

## 6. リスク・前提崩壊シナリオ

### 6.1 差額決済 10% が機能しないリスク
- 推し活ユーザーは「現地手渡し / 等価交換」志向
- 差額発生取引そのものが少ない可能性
- 緩和策: Phase 2 開始時に 5-8% で再検証可能

### 6.2 広告 UX が交換ツール文脈で機能しないリスク
- Linear/Aēsop 系ブランドポジショニングと AdMob は緊張関係
- 緩和策: Phase 2.5 で小規模 A/B、CTR/CPM データで投入判断
- ネイティブ広告 = 公演グッズ告知のみに絞る

### 6.3 Plus 課金率 1% 未達リスク
- 推し活市場での「コア機能課金」前例なし
- 緩和策: Phase 3 で A/B 検証、課金率 1% 未達なら即撤退

### 6.4 B2B 商談成立しないリスク
- バンダイ / サンリオ / HYBE は内製 (Weverse) 持ち
- 緩和策: Phase 4 まで PL に B2B 収益入れない、補助収益扱い

### 6.5 法規制 (資金移動業) 改正リスク
- Stripe Connect 収納代行スキームに改正影響の可能性
- 緩和策: Phase 2 開始前 6 ヶ月 (2026/Q4-2027/Q1) に専門家相談
